How uvpy.run works

Standalone Python tools, served as readable URLs.

uvpy.run hosts a curated collection of small Python scripts that you can inspect in the browser and run locally with uv run.

Each tool is designed to be self-contained: its Python version and dependencies live in the file through PEP 723 inline script metadata.

That means you can copy a command, let uv create the isolated environment, and avoid managing a project virtualenv for one-off utilities.

Running remote scripts carries risk. Review the source and declared dependencies before running a tool on your machine.

Why UV + PEP 723?

  • URL-based execution: scripts are downloaded and run locally.
  • Automatic dependencies: uv reads the script metadata and resolves packages for that run.
  • Isolated environments: each script can run without polluting your project environment.
  • Zero setup: no manual virtualenv or requirements file is needed for small tools.
